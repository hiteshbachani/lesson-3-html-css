Every programme is input, processing & output

Take out any programme and you'll find that there is some kind of input then it processes that input and finally you see the output.

Whenever you're building an app or any programme, it always help to think in terms of like what will be the input, processing & the output part of that programme or app

  • Breaking the programme into smaller chunks like this really helps in building big complex programmes

for eg: In the quiz app which you can play here
  1. it takes user's answers as input
  2. runs the processing that checks the answer with correct answer
  3. & finally it shows the output which is user's score in this case
